# Дизайн: баннер «Что тебя ждёт в клубе» (секция `#includes`)

## Что добавляем

Новая секция между блоком «Бизнес растёт…» (`#growth`) и блоком «Манифест» (`#manifesto`) в `index.html`. Источник — присланный пользователем референс-баннер (чёрный фон, красные размытые пятна по углам, красная плашка-бейдж слева сверху, красная плашка-кнопка справа снизу).

Эта секция — итоговая реализация пункта исходной спеки (`CLAUDE.md`) «Что входит в клуб». Текстовый список из 11 пунктов (встречи, мастермайнды, закрытый чат и т.д.) **не реализуется сейчас** — это отдельная задача на будущее. Вместо текста — горизontalная карусель из 6 фото без подписей: визуальное, атмосферное представление того же смысла.

**Порядок секций на странице после изменения:**

```
Hero → Основатель → Growth («Бизнес растёт…») → Includes (новая) → Манифест → Формат клуба → FAQ → финальный CTA
```

## Контент

- Бейдж/заголовок секции (одновременно `<h2>` для SEO, см. §6 `CLAUDE.md`): «Что тебя ждёт в клубе».
- Карусель из 6 фотографий клубной атмосферы, без подписей и текста на слайдах.
- Кнопка «Стать участником» — открывает существующую модалку заявки (`data-open-modal`), как и все остальные CTA на странице.

## Разметка

В отличие от остальных секций сайта, `.includes__inner` **не оборачивается в `.container`** — это сознательное исключение, нужное, чтобы карусель могла идти от края до края вьюпорта (bleed). `.includes` задаёт `position: relative` и служит координатной точкой для абсолютно спозиционированных бейджа и кнопки на десктопе; их отступ от края — `var(--space-3)`, то же значение, что использует `.container` для боковых полей, чтобы визуально бейдж/кнопка выглядели на одной линии с текстом в соседних секциях.

  <div class="includes__inner">

    <div class="includes__carousel" tabindex="0" role="region" aria-label="Фото из жизни клуба">
      <div class="includes__track">
        <div class="includes__slide"><img src="assets/club-photo-1.webp" alt="Атмосфера встречи клуба «ПОБЕДА»" loading="eager"></div>
        <div class="includes__slide"><img src="assets/club-photo-2.webp" alt="Атмосфера встречи клуба «ПОБЕДА»" loading="lazy"></div>
        <div class="includes__slide"><img src="assets/club-photo-3.webp" alt="Атмосфера встречи клуба «ПОБЕДА»" loading="lazy"></div>
        <div class="includes__slide"><img src="assets/club-photo-4.webp" alt="Атмосфера встречи клуба «ПОБЕДА»" loading="lazy"></div>
        <div class="includes__slide"><img src="assets/club-photo-5.webp" alt="Атмосфера встречи клуба «ПОБЕДА»" loading="lazy"></div>

Локальная красно-чёрная палитра задаётся на `.includes` так же, как на `.hero` и `.founder` (переопределение `--color-gold: #E31E24` и других переменных) — это уже установленный паттерн на сайте, третье применение, не новый.

## Визуальный стиль

- Фон секции: `#0A0A0C` (как `.hero`), с двумя размытыми красными пятнами по углам через CSS `radial-gradient` на `.includes__bg` — без новых файлов-картинок, тот же приём, что в `.hero__bg`/`.founder__bg`.
- `.includes__badge` — визуально красная плашка (`background: var(--color-gold)`, белый текст, `padding`, `border-radius: 2px`, `text-transform: uppercase`, `font-weight: 800`), но остаётся настоящим `<h2>` для разметки/SEO.
- `.includes__cta` — обычная `.btn.btn--gold` (красная заливка в локальной палитре секции), без новых модификаторов кнопки.
- Карусель полноширинная («bleed»): благодаря отсутствию `.container` на `.includes__inner` (см. выше) `.includes__carousel` естественно идёт от края до края вьюпорта, без `max-width`-ограничений — следующее фото заметно «выглядывает» за правый край экрана. Первый слайд получает `padding-left: var(--space-3)` (то же значение, что и боковое поле `.container`), чтобы не прилипать к самому краю экрана. Слайды — `aspect-ratio: 3 / 4` (как `.founder__photo`), фиксированная ширина (десктоп ~240px, мобильный ~62vw).
- Скроллбар карусели скрыт (`scrollbar-width: none`, `::-webkit-scrollbar { display: none }`) — подсказка о возможности скролла даётся «выглядывающим» краем фото и стрелкой, а не системным скроллбаром.

### Десктоп (≥768px)

Угловая композиция, повторяющая референс:
- `.includes__badge` — `position: absolute`, левый верхний угол секции.
- `.includes__cta` — `position: absolute`, правый нижний угол секции.
- Карусель — по центру секции вертикально, между бейджем и кнопкой.

### Мобильный (<768px)

Бейдж и кнопка уходят в обычный поток (не `absolute`). Так как `.includes__inner` без `.container`, оба получают свой `margin-left`/`margin-right: var(--space-3)` напрямую, чтобы выровняться по тем же боковым полям, что и текст в соседних секциях:
- `.includes__badge` — обычный блок сверху, по левому краю.
- Карусель — сразу под бейджем, минимальные вертикальные отступы (`padding` секции уменьшен относительно десктопной версии, чтобы карусель визуально доминировала, без «лишнего воздуха»).
- `.includes__cta` — обычный блок под каруселью, по центру (`margin: var(--space-3) auto`, `display: block` с фиксированной шириной по содержимому, не `btn--full` — кнопка не должна растягиваться на всю ширину экрана, как плашка с референса).

## Механика карусели (JS)

Новый блок логики в `js/main.js`, в том же стиле (ES5, `var`, без модулей/фреймворков), рядом с существующей логикой sticky-CTA:

- Горизонтальный скролл реализован через нативный `overflow-x: auto` + `scroll-snap-type: x mandatory` на `.includes__track`, `scroll-snap-align: start` на `.includes__slide`. Это даёт тач-свайп на мобильных без какого-либо JS.
- Для десктопа (мышь без трекпада) добавляется drag-to-scroll через `pointerdown`/`pointermove`/`pointerup` на `.includes__track` — курсор хватает и тащит карусель горизонтально.
- Кнопка `.includes__arrow--next` по клику скроллит на ширину одного слайда (`scrollBy({ left: slideWidth, behavior: 'smooth' })`).
- Кнопка `.includes__arrow--prev` скрыта атрибутом `hidden` изначально; слушатель `scroll` (passive) на `.includes__track` снимает `hidden`, когда `scrollLeft > 0`, и возвращает `hidden`, когда `scrollLeft` около `0`.
- Кнопка `.includes__arrow--next` скрывается аналогично, когда `scrollLeft + clientWidth >= scrollWidth` (карусель докручена до конца).
- Без автопрокрутки и без таймеров — целиком управляется пользователем (свайп, драг или клик по стрелке).

## Ассеты — фото для карусели

Файлы кладутся пользователем в:

```
assets/club-photo-1.webp
assets/club-photo-2.webp
assets/club-photo-3.webp
assets/club-photo-4.webp
assets/club-photo-5.webp
assets/club-photo-6.webp
```

Рекомендуемые пропорции — портрет **3:4** (как `assets/founder-photo.webp`), формат `.webp`. Порядок файлов = порядок показа в карусели. До появления реальных файлов `<img>` будет битым — это ожидаемое поведение на время плейсхолдера, как и для других фото-ассетов проекта (см. §8 `CLAUDE.md`).

## Навигация хедера

В `.site-nav` (десктоп) добавляется пункт «Что входит» со ссылкой на `#includes`. Итоговый порядок пунктов меню — `Основатель → Что входит → FAQ`, в порядке реального расположения блоков на странице (а не в порядке, подразумеваемом исходной спекой, где «Что входит» шёл раньше «Основателя» — порядок секций на странице с тех пор изменился).

Мобильный хедер не меняется (по `CLAUDE.md` — только лого + CTA, без гамбургер-меню).

## Вне скоупа

- Автопрокрутка карусели по таймеру.
- Текстовый список 11 пунктов «Что входит в клуб» — отдельная будущая задача, не пересекается с этим баннером.
- Подписи/заголовки на фото-слайдах.
- Новые юнит-тесты — секция не содержит чистых тестируемых функций (как и `.growth` ранее); существующие `tests/apply.test.js` и `tests/main.test.js` не затрагиваются.
